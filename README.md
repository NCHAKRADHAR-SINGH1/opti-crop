
OptiCrop is a lightweight web application that recommends the best crop for given soil and weather conditions. It accepts common agronomic inputs (nitrogen, phosphorous, potassium, temperature, humidity, pH and rainfall), scales them using a saved preprocessor, and returns a prediction from a pre-trained scikit-learn model.

## Features
- Simple Flask web UI (`templates/`) with a single prediction form
- Pre-trained model and scaler loaded from `models/model.pkl` and `models/scaler.pkl`
- Ready for deployment on Render using `Procfile` and `.python-version`

## Requirements
- Python 3.11 (pinned for Render compatibility)
- See `requirements.txt` for Python packages used by the app

## Quick start — run locally
1. Create a virtual environment and activate it:
   ```bash
   python -m venv venv
   # Windows
   venv\Scripts\activate
   # macOS / Linux
   source venv/bin/activate
   ```
2. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```
3. Ensure the trained model and scaler exist:
   - `models/model.pkl`
   - `models/scaler.pkl`
   (These should be created by your model training step — see `step7_evaluate_save.py`.)
4. Run the app for development:
   ```bash
   python app.py
   ```
   Then open http://127.0.0.1:5000

For production testing with Gunicorn:
```bash
gunicorn app:app
```

## Deploying to Render (recommended)
1. Push this repository to GitHub and connect it to Render.
2. Create a new **Web Service** in Render and link the `opticrop-agriculture` repo branch (e.g., `main`).
3. Use these settings in Render:
   - **Build Command:** `pip install -r requirements.txt`
   - **Start Command:** `gunicorn app:app`
   - **Environment:** set `PYTHON_VERSION` or add a `.python-version` file (this repo includes `.python-version` pinned to `3.11`).
4. If you run into a pandas build error, ensure Render uses Python 3.11 (we included `.python-version`) — older/newer Python versions sometimes require different binary wheels.

## Troubleshooting
- If a build fails during `pip install`, check the Render build logs. Common fixes:
  - Pin Python to `3.11` using `.python-version` or `PYTHON_VERSION` env var (already included).
  - Use prebuilt wheels by selecting a supported Python version.

## Run Order

### Epic 2: Data Collection and Analysis
```bash
python epic2_data_collection_analysis.py
# Covers: Import Libraries, Read Dataset, Univariate, Bivariate, Multivariate Analysis
# Saves:  univariate_analysis.png, bivariate_analysis.png, multivariate_analysis.png
```

### Epic 3: Data Pre-processing
```bash
python step1_null_check.py          # Story 1: Null values
python step2_outliers.py            # Story 2: Outliers + boxplot_outliers.png
python step3_seasonal_crops.py      # Story 3: Seasonal crops
python step4_train_test_split.py    # Story 4: Train/test split
```

### Epic 4: Model Building
```bash
python step5_kmeans_elbow.py        # Story 1: K-Means + elbow_plot.png
python step6_logistic_regression.py # Story 2: Logistic Regression
python step7_evaluate_save.py       # Story 3: Evaluate + saves model.pkl & scaler.pkl
python step8_predict_crop.py        # Story 4: Predict crop (run AFTER step7)
```

### Epic 5: Application Building
```bash
python app.py   # Run AFTER step7. Open http://127.0.0.1:5000
```
